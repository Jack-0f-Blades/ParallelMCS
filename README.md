# MCS – Maximum Clique Solver

Данный проект реализует алгоритмы поиска максимальной клики в неориентированном графе:
- **MCS** – последовательная версия (Tomita et al., 2010)
- **Parallel MCS** – параллельная версия с динамическим планированием задач
- **ILS (Iterated Local Search)** – эвристика для получения начальной нижней границы, может работать последовательно или параллельно

## Сборка
Для компиляции необходим компилятор с поддержкой C++17 и библиотекой pthread.

```bash
g++ -std=c++17 -O2 -pthread -DENABLE_LOGGING -o run_tests.exe \
    run_tests.cpp \
    Algorithm.cpp \
    MaxSubgraphAlgorithm.cpp \
    MCQ.cpp \
    StaticOrderMCS.cpp \
    MCS.cpp \
    CliqueColoringStrategy.cpp \
    SparseCliqueColoringStrategy.cpp \
    SparseMCS.cpp \
    SparseParallelMCS.cpp \
    OrderingTools.cpp \
    DegeneracyTools.cpp \
    GraphTools.cpp \
    Tools.cpp \
    ParallelMCS.cpp \
    ParallelRunner.cpp \
    TaskQueue.cpp \
    ILS_src/ILSSolver.cpp \
    Logger.cpp
```

## Параметры командной строки

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `--mode <mode>` | Режим работы: `mcs`, `mcs_par`, `ils_par+mcs`, `ils_par+mcs_par` | `mcs_par` |
| `--dir <path>` | Каталог с тестовыми графами (можно указать несколько) | отсутствует |
| `--input-file <file>` | Конкретный файл с графом | отсутствует |
| `--output <file>` | Файл для сохранения результатов | `results.txt` |
| `--timeout <sec>` | Тайм-аут на решение (в секундах) | `5400` |
| `--threads <N>` | Количество потоков для MCS (и ILS, если не задано отдельно) | число аппаратных ядер |
| `--queue-capacity <N>` | Ёмкость кольцевого буфера задач (`0` – автоматический выбор) | `max(200, n)` |
| `--split-depth <N>` | Глубина рекурсии, начиная с которой разрешается создание новых задач | `2` |
| `--target-queue-size <N>` | Целевой размер очереди при инициализации | `100` |
| `--low-threshold <N>` | Нижний порог очереди – при опускании ниже воркеры начинают генерировать задачи | `20` |
| `--nodes-between-checks <N>` | Частота проверки очереди (количество обработанных узлов) | `500` |
| `--min-task-size <N>` | Минимальный размер множества кандидатов, при котором задача может быть разделена | `10` |
| `--ils-scans <N>` | Количество итераций ILS на одну попытку | `1000` |
| `--ils-attempts <N>` | Количество независимых попыток ILS | `15` |
| `--ils-threads <N>` | Количество потоков для параллельного ILS | значение `--threads` |
| `--seed <N>` | Базовое зерно для генератора случайных чисел | случайное |
| `--log` | Включить детальное логирование (файл `mcs_log.txt`) | выключено |

## Режимы работы

mcs – последовательный MCS, без ILS.

mcs_par – параллельный MCS, без ILS.

ils_par+mcs – последовательный MCS с предварительным параллельным ILS (улучшение нижней границы).

ils_par+mcs_par – параллельный MCS с предварительным параллельным ILS.

Примечание о потоках: Параметр --threads указывает общее количество потоков, которое будет использовано. Для параллельного MCS один поток является мастером, остальные – воркерами. Поэтому, если вы хотите, например, 4 воркера, укажите --threads=5. Аналогично для ILS: при --ils-threads=5 один поток будет мастером, остальные 4 – воркерами.

## Входные данные 

Программа читает графы в формате DIMACS (расширения .clq, .graph, .txt):

```bash
c комментарий
p edge n m
e u v
```

## Примеры запуска

Запуск на одном файле в параллельном режиме:

```bash
./run_tests --mode mcs_par --input-file data/mygraph.clq --threads 5 
```

Запуск на всех графах из каталога с предварительным ILS:

```bash
./run_tests --mode ils_par+mcs_par --dir data/ --threads 9 --ils-attempts 10 --output results.txt
```

Включение логирования:

```bash
./run_tests --mode mcs --dir graphs/ --log
```

## Выходные данные

Результаты записываются в файл, указанный параметром --output (по умолчанию results.txt).
